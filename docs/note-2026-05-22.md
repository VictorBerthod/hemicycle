## Contexte
Le squelette frontend etait pret (Phases A-F) mais les pages affichaient des donnees plates : nom, groupe, circo. Objectif de la session : poser les fondations pour des **profils politiques riches** â€” distinguer parti politique et groupe parlementaire, permettre l'ajout manuel de tags editoriaux (poste, commission, profession, etc.) et exposer tout ca via l'API pour brancher le frontend ensuite.

## Travail realise

### Bloc 1 â€” Extension du modele de donnees
- Nouvelle table `Tag` (id, slug, libelle, categorie, couleur, description)
- Categories typees : `parti`, `poste`, `commission`, `profession`, `mandat`, `libre` (constante `TAG_CATEGORIES`)
- Table d'association `DeputeTag` (depute_id, tag_id, source) â€” le champ `source` distingue `etl-mapping`, `manuel`, `wikidata`...
- Champs profil ajoutes sur `Depute` : `profession`, `mandats_anterieurs`, `bio_short`
- Relation many-to-many `Depute.tags` (chargement `selectin` pour eviter le N+1)

### Bloc 2 â€” Schemas Pydantic
- `TagOut`, `TagCreate`, `TagUpdate`, `TagAttach`
- `DeputeListItem` et `DeputeDetail` exposent desormais `tags: list[TagOut]`
- `DeputeDetail` expose les trois champs profil enrichis

### Bloc 3 â€” Endpoints API
Router `/api/tags` (CRUD) :
- `GET /api/tags?categorie=parti&search=â€¦`
- `POST /api/tags` (slug auto-genere depuis libelle + categorie)
- `PATCH /api/tags/{id}` / `DELETE /api/tags/{id}`
- `GET /api/tags/categories` (comptage par categorie)

Router d'attachement (monte sur `/api/deputes`) :
- `POST /api/deputes/{uid}/tags/{tag_id}` body `{source: "manuel"}` â†’ renvoie la liste a jour
- `DELETE /api/deputes/{uid}/tags/{tag_id}` â†’ 204

Filtres etendus sur `GET /api/deputes` :
- `?tag=slug` (parametre repetable, AND multi-tags)
- `?tag_categorie=parti` (tous les deputes ayant au moins un tag de cette categorie)

CORS elargi pour autoriser POST/PATCH/DELETE depuis `localhost:5173`.

### Bloc 4 â€” Migration non destructive
`etl/migrate_add_profile_cols.py` ajoute les trois colonnes via `ALTER TABLE` SQLite sans toucher aux donnees existantes. Idempotent.

### Bloc 5 â€” Seed de la taxonomie
`etl/seed_tags.py` injecte :
- **14 partis** (Renaissance, MoDem, Horizons, UDI, LR, PS, EELV, PCF, LFI, Place publique, RN, Reconquete, UDR, Divers)
- **9 postes** parlementaires standards (president groupe, rapporteur general, ministre, etc.)
- **8 commissions** permanentes de l'AN

Puis auto-attache un tag parti a chaque depute via un mapping `groupe.acronyme -> parti`. Couverture finale : **601/601 deputes ayant un groupe** tagges automatiquement.

Mapping initial corrige en cours de session :
- `LFI-NFP` (acronyme reel 17e legislature) â†’ `parti-lfi`
- `UDDPLR` (Union des droites pour la Republique) â†’ `parti-udr`
